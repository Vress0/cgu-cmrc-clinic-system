# 專案需求分析

## 系統定位

「長庚大學中國醫學研究社義診健康紀錄系統」是義診現場使用的健康紀錄與流程管理系統，不是正式醫院 HIS 或完整電子病歷系統。第一版目標是支援掛號、診間、藥局與管理員在同一場義診中即時協作。

## 核心問題

- 減少紙本紀錄與重複填寫。
- 讓掛號、診間、藥局即時掌握個案狀態。
- 保留必要健康背景、診察、用藥、調劑、發藥與稽核紀錄。
- 避免過度設計，不加入 AI 診斷、知識圖譜、微服務或大型醫院 HIS 功能。

## 第一版邊界

- 採前後端分離：Next.js + FastAPI。
- PostgreSQL 作為主要資料庫。
- JWT、Refresh Token 與 RBAC 在 Phase 2 實作。
- Phase 1 僅建立可執行骨架、健康檢查 API、Docker Compose 與 README。
- 所有醫療相關異常值僅提示重新量測與通知現場醫療人員，不自動診斷。

## 主要角色

- 掛號組：建立個案、健康背景、同意紀錄與本次掛號。
- 診間組：候診、叫號、問診、量測、診察與處置用藥單。
- 藥局組：調劑、第二人核對、發藥、庫存扣除與退回診間。
- 管理員：場次、帳號、權限、藥品、庫存、統計與稽核。

## MVP 驗收流程

管理員建立場次，掛號人員新增或搜尋個案並建立掛號；診間完成問診、量測與處置用藥；藥局完成調劑、核對、發藥與扣庫；系統將 Visit 標記完成並留下操作紀錄，日後可查看歷次摘要。
